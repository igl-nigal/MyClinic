**Functional Requirements:**



* Login \& Security: Users can create accounts, log in, and reset passwords.
* Data Entry: The system allows users to enter, save, and edit information.
* Recommendations: Users can receive recommended activities based on their health.
* Healthscore: The software generates a health score of the user.
* Permissions: Admins can delete data, while regular users can only view it.
* Booking: The system books an appointment for the user with a doctor if needed.
* Notify: Will notify emergency contact if needed.



**Non-Functional Requirements:**



* Speed: Pages must load in less than 2 seconds.
* Capacity: The system must handle 1,000 users at the same time without crashing.
* Reliability: The software must be online and working 99.9% of the time.
* Security: All personal user data must be encrypted.
* Mobile-Friendly: The interface must work clearly on both phones and desktop computers.
* Backup: The system must automatically save a backup of all data every 24 hours.
